## Your Week

**31 commits, +4.6k LOC, v0.1에서 v0.2 하루만에.**

프로젝트 초기화부터 GitHub Release 배포까지 17시간. gstack workflow 전체를 활용 -- /office-hours(디자인 문서), /plan-eng-review(아키텍처 리뷰), /investigate(버벅임 근본원인), /ship(PR+Release).

Agent Teams로 3명 병렬 구현(perf-worker, input-worker, core-worker). 10개 작업을 종속성 그래프로 관리.

### What went well

- DWM 하드웨어 가속으로 전환한 판단. AllowsTransparency의 소프트웨어 렌더링 한계를 발견하고, 클릭 통과 문제를 Hide/Show 토글로 깔끔하게 우회.
- 사용자 피드백에서 "그냥 껐다 키면 안돼?"라는 가장 단순한 해법을 끌어냄.
- 실시간 사용자 테스트, 즉시 수정 루프. 온보딩 위치, 글씨 크기, 트레이 아이콘까지 피드백 받고 바로 반영.

### Where to improve

- 빌드 캐시 문제로 30분 삽질. `taskkill.exe`로 이전 프로세스를 종료 안 하고 새 버전을 실행해서 구 버전이 돌아감. 항상 프로세스 종료, 클린 빌드, 실행 순서.
- Codex rescue에 의존했지만 실질적인 결과를 못 받음. 복잡한 WPF 렌더링 문제는 외부 모델보다 직접 조사가 빠름.
- single-file exe에 네이티브 DLL이 안 들어가는 걸 배포 후에야 발견. CI에서 빌드한 exe를 직접 다운로드 테스트하는 스텝이 필요.

## 3 Things to Improve

1. **배포 전 end-to-end 테스트 추가.** GitHub Release에서 다운로드, 실행, 온보딩, 기능 테스트까지 자동화. 네이티브 DLL 누락 같은 건 CI에서 잡아야 함.
2. **멀티모니터 테스트를 개발 초기부터.** 온보딩 위치 6번 수정은 낭비. 처음부터 VirtualScreen 좌표를 고려하면 2번이면 충분.
3. **프로세스 종료를 자동화.** 빌드 스크립트에 `taskkill` 포함시켜서 구 버전 실행 문제 방지.
